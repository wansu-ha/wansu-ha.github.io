01 — Overview

5~10칸 8색 노노그램을 풀어 화판을 모으고, 그 화판으로 자기 영지를 큐레이션한다. 푼 화판은 세밀한 그림으로, 안 푼 화판은 한 칸의 대표색으로만 드러나는 안개 시야.

React 실력을 끌어올리고 Node 풀스택 경험을 쌓으려, "막히는 층 없이 혼자 끝까지" 만드는 걸 목표로 잡았다. 알고리즘·게임 렌더·백엔드·인프라를 한 명이 전부 책임졌다 — 기획·아키텍처·기술 선택은 직접 결정하고, 세부 구현은 AI 페어로 가속했다. 지금 누구나 접속해 플레이할 수 있다.

노노그램 풀이 엔진과 출제 검증

유저가 만든 화판이 풀리는지·정답이 몇 개인지 서버가 판정한다. 라인별 후보를 생성(composition)해 행 백트래킹 + 열 제약 전파로 풀고, 라인 시뮬레이션으로 난이도 별점(1~5)·라벨(universal·maniac·reject)을 매긴다. 복수 풀이는 거부하지 않고 2~5개는 "연작"으로 흡수, 6개 이상만 거부 — 거부 시 1셀 자동수정을 제안하고 전역 중복은 정규화 SHA-256으로 막는다. 출제검증(PVS) p99를 CI 회귀 감지선으로 건다.

solveAll(hints, size, 6) // 1=단일 · 2~5=연작 · 6+=거부 (size 5~10) // 출제검증(PVS) p99 5×5<5ms · 10×10<100ms · CI 회귀 게이트

토러스 무한 월드 + Phaser 렌더

세계가 끝없이 이어지는 도넛(토러스) 평면이라, 거리·충돌을 항상 "짧은 경로"로 계산해야 한다. 순수 좌표 유틸을 코어에 두고 백엔드 충돌 검사와 프론트 렌더가 공유한다. 세계는 Phaser, UI는 React가 맡되 둘 사이를 인터페이스 하나로 격리했다.

torusDelta(95, 5, 100) === 10 // 직선 -90 대신 wrap 통과 +10 (짧은 경로)

군집 인식 연결망 — 도시가 스스로 뭉치다

영지들이 자연스러운 군집(도시)을 이루도록, 토러스-MST(Prim) 백본을 깔고 가까운 간선(rest ≤ dLocal)만 남겨 군집을 만든다. 군집과 군집 사이는 트리라서 다리가 정확히 하나, 군집 안은 Gabriel 이웃으로 촘촘하게 — 백본이 한 트리라 어느 배율에서 봐도 같은 구조가 유지된다. 코어 순수 함수를 Phaser 월드·미니맵이 그대로 호출한다.

torusMst(estates) → 군집 = rest ≤ dLocal 연결요소 // intra: Gabriel(같은 군집) ∪ bridges: 군집쌍당 1 // gameWorld.ts 라이브 렌더에서 실제 호출

협력 경제 — 장부와 동시성

이중 통화의 모든 변동을 장부 1행으로 원자적 기록(잔액은 캐시). 동시 차감의 더블스펜드는 가드를 차감과 한 쿼리로 묶어 막았다. 떠다니는 화판은 cron이 면적 비례로 자동 공급하되, 한 틱 처리량을 제한해 작은 서버가 멈추지 않게 했다 — 실제 장애를 겪고 고친 부분.

UPDATE player SET balance = balance - :amt WHERE balance >= :amt -- 0행이면 롤백
